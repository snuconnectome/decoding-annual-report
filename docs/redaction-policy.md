# Redaction Policy (공개 전 반드시 준수)

본 저장소는 **public** GitHub repo로 운영된다. 원본 3년차 연차보고서 PDF(2025-12-08 제출본, 98페이지)는 개인 식별정보·민감정보를 포함하므로, 아래 정책을 **엄격히** 준수하여 공개 버전을 생산한다.

## 핵심 원칙

1. **원본 보존**: 원본 PDF는 `source/` 디렉토리에 배치하되, `.gitignore`로 저장소에 **절대** 포함시키지 않는다.
2. **본문 미기재 원칙**: redaction 필요 정보는 `report/*.md`에 **애초에 기재하지 않는다** (1차 방어선).
3. **자동 스캔**: `scripts/redact-check.py`로 커밋 전 패턴 매칭 검증 (2차 방어선).
4. **수동 검토**: public push 직전 사람 리뷰 (3차 방어선).

## 자동 스캔 패턴 (scripts/redact-check.py 대상)

다음 정규표현식에 매칭되는 항목이 `report/**/*.md` 에 존재하면 빌드 실패한다.

```
# 한국 휴대전화
01[0-9][-\s]?\d{3,4}[-\s]?\d{4}

# 일반 전화 (지역번호 포함)
0\d{1,2}[-\s]?\d{3,4}[-\s]?\d{4}

# 주민등록번호 형식
\b\d{6}[-\s]?[1-4]\d{6}\b

# 국가연구자번호 / 사업자번호 / 법인번호 (8–13자리 순수 숫자)
\b\d{8,13}\b

# 사인 이미지 파일명 참조
(sign|signature|서명)\.(png|jpg|jpeg|gif)
```

일부 과제번호(`RS-YYYY-NNNNNNNN`)는 화이트리스트로 허용한다. 검토 후 `redact-check.py`의 `ALLOWED_PATTERNS`에 추가.

## 공개 전 체크리스트

- [ ] 원본 PDF가 `source/`에만 존재 (`git ls-files | grep '\.pdf$'` → 결과 없음)
- [ ] `scripts/redact-check.py report/` 실행 결과 매칭 0건
- [ ] `report/build/final.pdf` 열어 표지·메타 페이지 수동 확인
- [ ] `slides/images/*.png` 시각적 검토 — 전화·서명·번호 노출 여부
- [ ] `docs/`, `README.md`, `data/*.yaml`에서 민감정보 미포함 확인
- [ ] `git diff origin/main..HEAD`로 push 예정 내용 최종 리뷰

## 사고 대응 (민감정보 노출 발생 시)

1. 즉시 해당 커밋을 `git revert` 후 force push — **단순 revert로는 history에서 제거되지 않음을 인지**
2. `git filter-repo` 또는 `BFG Repo-Cleaner`로 해당 정보 포함 blob 완전 제거
3. 노출된 정보(예: 전화번호 유출 시)가 실제 민감 도용 가능하면 본인·기관 정보보안팀에 보고
4. `force push` 후 모든 fork/clone 사용자에게 알림 (issue pin)

**교훈**: 사후 대응은 매우 비용이 높다. 사전 예방 3중 방어선을 반드시 가동할 것.
